The generated project structure is as follows :

   │   .classpath     << used by eclipse
   │   .project       << used by eclipse
   │   pom.xml        << used by Maven for building, documenting, and analyzing the project
   │
   ├───.settings    << used by eclipse
   │       org.eclipse.core.resources.prefs    << used by eclipse
   │       org.eclipse.jdt.core.prefs          << used by eclipse
   │       org.eclipse.m2e.core.prefs          << used by eclipse
   │
   ├───src                                   << Source files directory
   │   ├───build                             << Files specific to the build process, not distributed 
   │   │   ├───conf
   │   │   │       buildNumber.properties     << Maintains the current build number
   │   │   │       checkStyle.xml             << Code analysis configuration file
   │   │   │
   │   │   └───dist                         << Unprocessed files that should be part of the project distribution
   │   ├───main                              << code files go under the main branch, followed by their specific language
   │   │   │                                  <<  If this project was perl, it would be src/main/perl, python would be /src/main/python 
   │   │   ├───java
   │   │   │   └───com.tabtonic                 << com.tabtonic.kerberosAREA is the package name
   │   │   │       └───folders
   │   │   │           └───kerberosAREA
   │   │   │                   Artifactid.java << We start out with an artifact class
   │   │   │                   Main.java       << and a Main (runnable) class
   │   │   │
   │   │   └───resources                     <<resources are external files leveraged by the code under main.
   │   │                                        <<  Things like images, configuration files, etc.
   │   │                                        <<  src/main/resources/** are distributed with the package
   │   │                                       
   │   ├───test                               << Test files directory
   │   │  ├───conf                           <<  Configuration files for testing
   │   │  ├───java                           << Just like under src, each language gets it's own directory  
   │   │  │   └───com.tabtonic                   <<  com.tabtonic.kerberosAREA is the package name
   │   │  │       └───folders               <<  packages should correspond to the packages they test under main
   │   │  │           └───kerberosAREA
   │   │  │                   ArtifactidTest.java
   │   │  │                   MainTest.java
   │   │  │
   │   │  └───resources                      << resources specific to running tests
   │   │          log4j.xml                     << logging configuration for tests
   │   ├───site                               << the project site files
   │       │   site.xml                         << configuration file for site appearance/structure
   │       │
   │       ├───fml                            << FAQ Meta Language files (processed into html during mvn:site)
   │       │       faq.fml                      << Frequently Asked Questions
   │       │
   │       ├───markdown                       << Markdown files
   │       │   ├───doc
   │       │   │       index.md
   │       │   │       
   │       │   └───info                      << project startup documentation
   │       │           default-plugins.md
   │       │           logging-with-log4j.md
   │       │           logging-with-logback.md
   │       │           logging-with-slf4j.md
   │       │           project-setup.md
   │       │           project-structure.md
   │       │           working-with-maven.md
   │       │
   │       ├───resources                    << resources are copied into the site without processing
   │       │   │
   │       │   └───images                  
   │       │           logo-tabtonic-150x75.png
   │       │
   │       └───xhtml                        << sample xhtml page
   │               using-xhtml.xhtml
   │    
   │
   ├───target                               << target directory will be regularly deleted, contains the results of the build process
   │   ├───classes
   │   │   ├───com.tabtonic
   │   │   │   └───folders
   │   │   │       └───kerberosAREA
   │   │   │               Artifactid.class
   │   │   │               Main.class
   │   │   │
   │   │   └───META-INF
   │   │       │   MANIFEST.MF
   │   │       │
   │   │       └───maven
   │   │           └───com.tabtonic
   │   │               └───kerberosAREA
   │   │                       pom.properties
   │   │                       pom.xml
   │   │
   │   └───test-classes
   │       │   log4j.xml
   │       │
   │       └───com.tabtonic
   │           └───folders
   │               └───kerberosAREA
   │                       ArtifactidTest.class
   │                       MainTest.class
   │
   └───tmp                  << Regularly cleaned, not processed, not stored in SCM
           └───log          << Output folder for testing log results
