Using XHTML for project pages

Markdown is easy to work with, but there are some things that Markdown is not very good at.

Use XHTML when you need to

  • Insert Images
  • Use Tables
  • Use Javascript
  • Do complicated layouts

You can build the xhtml page in Dreamweaver or any other XHTML editor.

You can use XHTML, Markdown, FML, LaTeX, RTF, or Xdoc to produce site pages as long as they are in the appropriate directory.

If a file has the extra extension ".vm" it will be rendered by Velocity. This allows you to include things like the Project Name, Version, etc. in your site pages.
